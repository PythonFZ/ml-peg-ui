## Unsatisfied Requirements (Blockers)

### FR-2.2: Multi-entry autocomplete for filtering by benchmark name

**Phase:** 5 | **Verification:** BLOCKED

ROADMAP success criterion #2: "Typing in the benchmark filter autocomplete narrows the visible columns; multiple benchmarks can be selected simultaneously."

Implementation provides a single-select navigation autocomplete ("Go to benchmark...") that calls `router.push()` and clears. This satisfies quick-navigation but not the multi-entry column-narrowing filter.

**To close:** Add multi-select Autocomplete driving `columnVisibilityModel`, or update FR-2.2 to accept the navigation UX.

## Cross-Phase Integration Issues

### Critical: MinioBackend.list_keys Contract Mismatch

**Affects:** NFR-2.3, FR-3.4 | **Severity:** Production bug

`FilesystemBackend.list_keys` returns basenames (`["37Conf8", "file.json"]`). `MinioBackend.list_keys` returns full object keys (`["conformers/37Conf8"]`). Callers in `api/index.py` (`_build_category_index`, `benchmark_figures_index`, `neb_frames`) assume basenames — causing double-prefix paths in production.

**Impact:** All category/figure/NEB routes silently fail against MinIO. Local dev works fine.

### Minor: li_migration Dead NEB Viewer

**Affects:** FR-6.4 | **Severity:** Minor

`CUSTOM_VIEWERS` maps `li_migration: 'neb'` but `data/nebs/li_migration/` does not exist. NEB viewer tab appears but returns 404 for every model.
