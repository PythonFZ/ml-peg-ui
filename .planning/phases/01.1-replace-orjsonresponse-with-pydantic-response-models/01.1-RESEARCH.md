## Summary

This phase is a focused, mechanical refactor of the FastAPI backend. The `envelope()` helper function in `api/index.py` currently returns plain `dict` objects wrapped in `{"data": ..., "meta": {...}}`. All 5 endpoints use `response_class=ORJSONResponse` and the FastAPI app is constructed with `default_response_class=ORJSONResponse`. The `orjson` library is used in both `FilesystemBackend.get_json` and `MinioBackend.get_json` in `api/storage.py`.

Pydantic v2 (v2.12.5 is installed) and FastAPI v0.135.1 are the versions in use. FastAPI's native JSON serialization via Pydantic return type annotations is both simpler and faster than `ORJSONResponse` for structured data. `json.loads` (stdlib) accepts both `str` and `bytes`, making it a drop-in replacement for `orjson.loads` in the storage layer.

The scope is exactly 4 files: `api/models.py` (new), `api/index.py` (refactor), `api/storage.py` (replace import), `pyproject.toml` (remove dependency), and updates to `tests/test_api.py`.

**Primary recommendation:** Create `api/models.py` first, then update `api/index.py` endpoints one-by-one to use the new models, then clean up storage and dependency.

### Pattern 1: Pydantic model with `extra="allow"` for dynamic columns

**What:** `BenchmarkRow` has typed base fields plus unlimited extra fields that pass through without validation.

**When to use:** When a subset of fields is known and typed, but the full set varies per benchmark.

**Example:**
```python
# Verified locally against pydantic 2.12.5
from pydantic import BaseModel, ConfigDict

class BenchmarkRow(BaseModel):
    model_config = ConfigDict(extra="allow")
    id: str
    MLIP: str
    Score: float | None = None

row = BenchmarkRow.model_validate({"id": "x", "MLIP": "y", "Score": 0.9, "custom_metric": 1.23})
# model_dump() -> {"id": "x", "MLIP": "y", "Score": 0.9, "custom_metric": 1.23}
```

FastAPI serializes extra fields correctly when the endpoint return type is `BenchmarkTableResponse`. Verified.

### Pattern 2: FastAPI return type annotation for auto-serialization

**What:** Set Python return type on the async function. FastAPI calls `.model_dump()` and serializes to JSON without any `response_class` argument.

**When to use:** Every endpoint that returns a Pydantic model instance.

**Example:**
```python
# Verified locally against FastAPI 0.135.1
from api.models import HealthResponse

@router.get("/health")
async def health() -> HealthResponse:
    return HealthResponse(status="ok")
# Returns: 200 {"status": "ok"} with Content-Type: application/json
```

No `response_class=ORJSONResponse` and no `response_model=` kwarg needed when the Python type annotation is set.

### Pattern 3: Composing envelope shape from Meta + data list

**What:** Each list endpoint returns a model with `data: list[X]` and `meta: Meta`.

**Example:**
```python
# api/models.py
from pydantic import BaseModel

class Meta(BaseModel):
    count: int
    columns: list[str] | None = None

class BenchmarkTableResponse(BaseModel):
    data: list[BenchmarkRow]
    meta: Meta

# api/index.py (replaces envelope() call)
rows_validated = [BenchmarkRow.model_validate(r) for r in rows]
return BenchmarkTableResponse(
    data=rows_validated,
    meta=Meta(count=len(rows_validated), columns=columns),
)
```

### Pattern 4: `json.loads` as drop-in for `orjson.loads`

**What:** Python stdlib `json.loads` accepts both `str` and `bytes` since Python 3.6+.

**Example:**
```python
# storage.py â€” FilesystemBackend
import json
return json.loads(full_path.read_bytes())  # bytes input: works

# storage.py â€” MinioBackend
return json.loads(response.read())  # bytes input: works
```

### Pattern 5: Test validation via `model_validate`

**What:** After `response.json()`, pass the dict to `Model.model_validate()`. If the schema drifts from what the endpoint actually returns, Pydantic raises `ValidationError`.

**When to use:** Replaces raw dict key assertions in tests.

**Example:**
```python
from api.models import BenchmarkTableResponse

def test_benchmark_table(test_client):
    response = test_client.get("/api/v1/benchmarks/37conf8/table")
    assert response.status_code == 200
    body = BenchmarkTableResponse.model_validate(response.json())
    assert len(body.data) > 0
    assert body.meta.count == len(body.data)
```

Recommendation: use `model_validate` (not `model_construct`) in tests. `model_validate` runs validation and will catch schema drift. `model_construct` skips validation and would defeat the purpose.

### Anti-Patterns to Avoid

- **Mixing `response_class` with return type annotation:** Redundant; FastAPI uses the return type annotation. Drop all `response_class=ORJSONResponse` parameters.
- **Keeping `default_response_class=ORJSONResponse` on the app:** Remove this from the `FastAPI(...)` constructor.
- **Using `model_construct` in tests:** Skips validation, won't catch schema drift.
- **Importing `ORJSONResponse` anywhere after this phase:** The import `from fastapi.responses import ORJSONResponse` must be removed from `api/index.py`.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Dynamic column passthrough | Custom dict-merging serializer | `ConfigDict(extra="allow")` | Pydantic handles extra fields natively in v2 |
| OpenAPI schema for response shape | Manual `json_schema_extra` | Return type annotation | FastAPI auto-generates schema from Pydantic model |
| JSON deserialization | Custom parser | `json.loads` (stdlib) | Handles bytes, str, and all edge cases |

## Common Pitfalls

### Pitfall 1: Forgetting to remove `default_response_class` from the app constructor

**What goes wrong:** The app still nominally registers `ORJSONResponse` as default even after all endpoints are updated. The `orjson` import in `api/index.py` remains, so removing the package still breaks at import time.

**How to avoid:** After updating all endpoints, search `api/index.py` for any remaining `ORJSONResponse` reference including the `FastAPI(...)` constructor call and the import statement.

**Warning signs:** `ImportError: cannot import name 'ORJSONResponse'` after `uv remove orjson`.

### Pitfall 2: `json.loads` vs `orjson.loads` on `None` or empty bytes

**What goes wrong:** `orjson.loads(b"")` raises `JSONDecodeError`. `json.loads(b"")` raises `JSONDecodeError`. They behave identically on degenerate input â€” no behavior change here, but verify no code path passes empty bytes.

**How to avoid:** No change needed â€” behavior is identical. The existing try/except in `models` and `benchmark_table` endpoints already handles downstream errors.

### Pitfall 3: Pydantic v2 `model_dump()` vs `.dict()` in test assertions

**What goes wrong:** Old Pydantic v1 code used `.dict()`. Pydantic v2 uses `.model_dump()`. Since this project already uses Pydantic v2, use `.model_dump()`.

**How to avoid:** Always use `model_validate` and `model_dump` (v2 API). Verify no `.dict()` calls are introduced.

### Pitfall 4: `BenchmarkRow` extra fields and `response_model` validation stripping

**What goes wrong:** If a `response_model=` kwarg were specified on an endpoint, FastAPI would filter the response through that model and strip extra fields. But since we use return type annotations (not `response_model=`), extra fields are preserved.

**How to avoid:** Never add `response_model=BenchmarkTableResponse` to the `@router.get(...)` decorator. Use only the return type annotation `-> BenchmarkTableResponse`.

```python
from __future__ import annotations

from pydantic import BaseModel, ConfigDict


class Meta(BaseModel):
    count: int
    columns: list[str] | None = None


class HealthResponse(BaseModel):
    status: str


class BenchmarkEntry(BaseModel):
    slug: str
    name: str


class CategoryItem(BaseModel):
    slug: str
    name: str
    benchmarks: list[BenchmarkEntry]


class CategoriesResponse(BaseModel):
    data: list[CategoryItem]
    meta: Meta


class BenchmarkRow(BaseModel):
    model_config = ConfigDict(extra="allow")
    id: str
    MLIP: str
    Score: float | None = None


class BenchmarkTableResponse(BaseModel):
    data: list[BenchmarkRow]
    meta: Meta


class ModelEntry(BaseModel):
    id: str
    display_name: str


class ModelsResponse(BaseModel):
    data: list[ModelEntry]
    meta: Meta
```

### Endpoint refactor pattern (categories example)

```python
# Before
@router.get("/categories", response_class=ORJSONResponse)
async def categories(request: Request, response: Response) -> dict:
    response.headers["Cache-Control"] = CACHE_HEADER
    return envelope(request.app.state.categories)

# After
from api.models import CategoriesResponse, CategoryItem, Meta

@router.get("/categories")
async def categories(request: Request, response: Response) -> CategoriesResponse:
    response.headers["Cache-Control"] = CACHE_HEADER
    cats = request.app.state.categories
    return CategoriesResponse(
        data=[CategoryItem.model_validate(c) for c in cats],
        meta=Meta(count=len(cats)),
    )
```
