  <name>Task 1: Create color, format, and model-links utility modules</name>
  <files>src/lib/color.ts, src/lib/format.ts, src/lib/model-links.ts</files>
  <action>
1. Create src/lib/color.ts:
   - `import { interpolateViridis } from 'd3-scale-chromatic'`
   - `normalizeScore(value: number, good: number, bad: number): number` — compute `(value - bad) / (good - bad)` clamped to [0, 1]. Handle `good === bad` by returning 0.5.
   - `viridisR(normalizedScore: number): string` — return `interpolateViridis(clamped)` where clamped is `Math.max(0, Math.min(1, normalizedScore))`. NOTE: d3's interpolateViridis(0) = purple, (1) = yellow. Since normalizedScore 1.0 = good, passing it directly gives yellow=good, which IS viridis_r behavior.
   - `textColorForViridis(normalizedScore: number): 'black' | 'white'` — return `'black'` for scores > 0.4, `'white'` otherwise. This ensures WCAG AA contrast on viridis backgrounds (NFR-4.2).

2. Create src/lib/format.ts:
   - `formatSigFigs(value: number | null | undefined, sigFigs = 3): string` — return `'--'` for null/undefined/NaN/Infinity, otherwise `value.toPrecision(sigFigs)`. Use em-dash for missing values.

3. Create src/lib/model-links.ts:
   - Export `MODEL_LINKS: Record<string, string | undefined>` — a static mapping of model id to GitHub/HuggingFace URL.
   - Populate with known models: `mace-mp-0a` -> ACEsuit/mace-mp, `mace-mp-0b3` -> ACEsuit/mace-mp, `mace-mpa-0` -> ACEsuit/mace-mp, `mace-omat-0` -> ACEsuit/mace-mp, `orb-v3-consv-inf-omat` -> orbital-materials/orb-models, etc.
   - For unknown models, use `undefined` — the table should not render a broken link.
   - Add a comment: "// Update this map when new models are added to the benchmark suite"
